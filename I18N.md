# i18n (English now, Macedonian later)

## Approach
- Build UI in English.
- Wrap all UI text in a translation helper from day 1.
- Use a locale-aware route structure so adding mk later is just adding translation files.

## File structure (recommended)
- src/i18n/
  - en.json
  - mk.json
  - index.ts (t() helper)

## Translation rules
- Never hardcode UI strings in components.
- Always use `t("key.path")`.
- Start with en.json filled; mk.json can be empty placeholders.
