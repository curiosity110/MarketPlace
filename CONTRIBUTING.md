# Contributing / Operating Rules

## Non-negotiable rules
- No binaries in git.
- No secrets in git.
- All changes must be small and reviewable.
- Update TASKS.md when finishing a feature.

## Branching
- feature/<name>
- Merge via PR. Keep PRs under ~400 lines when possible.

## Commit messages
- feat: ...
- fix: ...
- chore: ...
- docs: ...
